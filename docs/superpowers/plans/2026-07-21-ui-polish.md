# UI Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver the approved Kool Studio navigation, motion, responsive-layout, project-localization, interaction, and social-preview refinements without changing the brand system.

**Architecture:** Extend the current components and data model. Keep layout ownership local: `Navbar` owns the brand orb, `ImageStrip` owns homepage hero positioning, `ProjectHero` owns both its fixed image and matching flow spacer, and `FooterBar` owns its separator/safe-area surface. Use the canonical project catalog plus an explicit display-order list rather than physically moving large project records.

**Tech Stack:** Next.js 16 App Router, React 19, TypeScript, Tailwind CSS v4, Framer Motion 12, Swiper 14, next-intl, native Node.js test runner.

## Global Constraints

- Preserve Poppins, `#E5DDD0` beige, `#FC3117` coral, `#1A1A1A` dark, and the existing information architecture.
- Use `@/i18n/navigation` for internal links and `useTranslations()` for public copy.
- Keep Polish and English message keys in parity.
- Do not add a UI library, animation package, typeface, or color.
- Keep the private design-system route out of navigation and sitemap.
- Use `$CONDUCTOR_PORT` for local development.
- Respect `prefers-reduced-motion`.
- Run `pnpm check` and browser QA before handoff.
